********************************* 项目介绍 **********************************
项目名称：基于python的网络端口扫描器
编写语言：python
代码量：150行左右
最后更新时间：2019年3月18日
项目介绍：用python编写的端口扫描器，使用TCP connect()方式扫描，并且可以将结果导出到链接的MySQL数据库中

********************************** 运行步骤 *********************************
    - 在第一次使用该软件时，先进行如下配置：（仅在使用数据库时才需要）
        1. 打开dbcfg.txt文件，将[database]中的各项数据更改为自己的账户数据
	- dbhost 为数据库所在的ip地址，本机默认为'127.0.0.1'
	- dbport 为mysql占用的端口
	    ·可使用 show global variables like 'port'; 进行查看
	- dbname 为要使用的数据库名称
	- dbuser 为登录mysql时使用的用户名
	- dbpassword 为登录mysql时使用的密码
	- dbcharset 是数据库使用的编码，默认为utf-8

        2. 在版本高于2.2.4之后不需要第2步了！
	进入mysql，在要使用的数据库内新建一个表，指令如下:
	> drop table if exists results;
	> create table results(
			id int not null primary key auto_increment,
			port int not null,
			ip varchar(255) not null,
			scan_time timestamp not null
			);

        3. 版本2.2.5中修复了此问题
       打开PortScanner.exe，勾选“导入到数据库”之后点击确认，记录报错信息中显示的路径（如果没有报错更好(｀・ω・´)）

        4. 将写好的配置文件dbcfg.txt转移到第3步中显示的路径中，并建议在本文件夹下留一个快捷方式以便以后更改

    - 使用方法：打开PortScanner.exe，选择合适的搜索方式后填入ip和对应的端口号，点击确认开始搜索。在搜索结束后结果会显示在右侧的输出栏（以及对应的数据库）
    - 又或者可以从gui.py中运行主函数进入程序界面，这样的话是不需要更改配置文件的路径的
    // 注：方法中的“对全端口进行扫描”耗时在2分钟左右，请耐心等待

********************************** 项目构成 *********************************
    |----README.txt			// 帮助文件，你现在读的文档
    |----config.py			// 读取配置文件的模块
    |----dbconnect.py		// 与mysql数据库链接的模块
    |----portScanner.py		// 对ip端口进行扫描的模块
    |----gui.py			// 实现程序界面的模块
    |----win_set.py                                    // 将某个指定窗口设置在屏幕中央
    |----dbcfg.txt			// 链接数据库的配置文件
    |----PortScanner.exe		// 生成的可执行程序

    // 注:各个模块的测试程序在当前模块的main函数中
